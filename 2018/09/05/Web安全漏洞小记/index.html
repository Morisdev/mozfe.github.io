Web安全漏洞小记

Sep 5, 2018

前情提要：昨天ISC互联网安全大会现场，我们一个退市近2年的产品被发现了安全漏洞。于是今天早晨，当我刚接满一杯咖啡，准备逛逛V2EX的时候，一批Jira工单甩到了我头上。我也有一年多没有维护过这个项目了，不得不说有一些焦虑。
尽管晚上的时候已经修复的差不多了，还是想在这儿记录一下有趣的细节。希望在以后的编码过程中更多的考虑Web安全。

OS命令注入

在网络模块，多个配置功能都存在OS命令注入的漏洞。形式如下：

注意参数中的反引号，PHP会尝试将反引号中的内容当做shell命令来执行。当这个请求当中的address参数被拼接到一个命令中，由PHP的exec()或system()函数执行时，非法命令即被渗透到系统中。
我看了一下代码，发现address参数已经使用escapeshellarg转义过了，但是单独使用这个方法，当使用单引号或反引号尝试攻击时是无法禁止的，应结合使用escapeshellarg + escapeshellcmd确保字符串完全被转义。参考此篇。

反射性XSS

攻击形式是修改请求头的Content-Type，比如从”application/json”改为”text/plain”，会造成反射性XSS。修复方案为在统一的控制器入口里，强制设定响应的Content-Type。

Burp Suite Pro

修复这些安全漏洞的过程中，都使用了一款很酷的工具，Burp Suite Pro，它是一个集成了抓包、处理HTTP、爆破等各种web安全工具的平台。
使用技巧如下：

  1. 在浏览器中设置代理
  2. 在Burp Suite Pro的Proxy-Options选项卡里配置代理
  3. 打开web应用，在Proxy-Intercept选项卡里监听HTTP消息
  4. 复制监听到的HTTP消息，右键，选择Send to Repeater
  5. 在Repeater选项卡里进行web攻击
